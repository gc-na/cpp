# C++에서의 catch: 예외 처리의 핵심

## 개요
C++에서 `catch`는 예외 처리 메커니즘의 중요한 요소로, 코드 실행 중 발생할 수 있는 예외 상황을 관리하는 데 사용됩니다. `catch` 블록은 `try` 블록과 함께 사용되어, 예외가 발생했을 때 해당 예외를 처리하는 역할을 합니다.

## 문서화
### 목적
`catch`는 C++에서 예외를 처리하기 위한 구문으로, 프로그램의 비정상적인 종료를 방지하고 안정성을 높이는 데 기여합니다. 예외가 발생할 경우, 적절한 오류 처리를 통해 프로그램 흐름을 유지할 수 있습니다.

### 사용법
`catch` 구문은 `try` 블록과 함께 사용되며, 다음과 같은 형식을 갖습니다:

```cpp
try {
    // 예외가 발생할 수 있는 코드
} catch (예외타입 &e) {
    // 예외 처리 코드
}
```

여기서 `예외타입`은 처리할 예외의 유형을 정의하며, `e`는 해당 예외 객체를 참조합니다. 여러 개의 `catch` 블록을 사용하여 다양한 예외를 처리할 수 있습니다.

### 세부 사항
- **여러 catch 블록**: 여러 종류의 예외를 처리하기 위해 여러 개의 `catch` 블록을 사용할 수 있습니다.
- **다형성**: 기본 클래스 포인터를 사용하여 파생 클래스의 예외를 처리할 수도 있습니다.
- **예외 전파**: `catch` 블록 내에서 예외를 다시 던질 수 있으며, 이는 상위 호출자에게 예외를 전달하는 데 유용합니다.

## 예제
다음은 `catch`를 사용하는 간단한 예제입니다.

```cpp
#include <iostream>
#include <stdexcept>

void mayThrow() {
    throw std::runtime_error("오류 발생!");
}

int main() {
    try {
        mayThrow();
    } catch (const std::runtime_error& e) {
        std::cout << "예외 처리: " << e.what() << std::endl;
    }
    return 0;
}
```

위 코드는 `mayThrow` 함수에서 발생한 예외를 `catch` 블록에서 처리하여, 프로그램이 종료되지 않고 오류 메시지를 출력합니다.

## 설명
### 일반적인 실수 및 주의사항
- **catch 블록을 놓치는 경우**: 예외가 발생했는데 `catch` 블록이 없으면 프로그램이 비정상적으로 종료됩니다.
- **catch 블록의 순서**: 파생 클래스의 예외는 기본 클래스 예외보다 먼저 처리해야 합니다. 그렇지 않으면 기본 클래스 `catch`가 모든 예외를 처리하여 파생 클래스의 예외를 잡지 못할 수 있습니다.
- **예외의 재던지기**: `throw;`를 사용하면 현재의 예외를 다시 던질 수 있습니다. 이는 예외 처리를 위임할 때 유용합니다.

## 한 줄 요약
C++의 `catch`는 예외 처리를 통해 프로그램의 안정성을 높이는 필수 요소입니다.
